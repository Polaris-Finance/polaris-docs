# Polaris Safety & Verification

<PageStatusBanner lastUpdated="June 8, 2026" />

Use this checklist before connecting a wallet, approving a token, or signing a transaction.

## Verification checklist

| Check | Required source |
|---|---|
| Current phase and official app | [Launch Status](/launch-status) |
| Public Testnet 1 quickstart | [Public Testnet Quickstart](/quickstart) |
| Network and chain ID | [Launch Status](/launch-status) |
| Contract addresses | [Contracts & Addresses](/resources/contracts) |
| Numeric parameters | [Parameters](/resources/parameters) |
| Testnet manifest paths | [Public Testnet 1 Artifacts](/resources/testnet) |
| Risk model | [Risk Disclosure](/resources/risk-disclosure) |
| Audit, bounty, and disclosure status | [Audits & Security](/resources/audits-security) |

## Things Polaris will not ask for

- Your seed phrase, private key, or recovery words.
- A signature from an unofficial frontend.
- Mainnet funds for Public Testnet 1.
- A production address before [Launch Status](/launch-status) says production is live.
- A bug-bounty or disclosure form not linked from [Audits & Security](/resources/audits-security).

## Wallet prompt checks

Before signing, verify:

1. The app URL is the official URL for the current phase.
2. The wallet network is the intended network.
3. The token, amount, spender, and receiving contract match the action.
4. The address appears on [Contracts & Addresses](/resources/contracts).
5. The transaction matches the action tab you are using.

If any item fails, reject the signature and use [Troubleshooting](/troubleshooting) before trying again.
