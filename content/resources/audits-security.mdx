# Polaris Audits & Security

> **Pre-launch note:** Audits and verifications are ongoing; this page will be updated with major developments.


## The security model

Polaris's intended security model rests as much on what it *cannot* do as on what it can:

- **Immutability.** The core contracts are intended to be non-upgradeable after deployment. This removes upgrade-key and proxy-admin attack surface, raising the stakes on getting the code right: immutability cuts both ways — it cannot be tampered with, and it cannot be patched.
- **No admin keys.** The design has no freeze, blacklist, or pause functions for anyone, including the team. No privileged key is intended to move user funds.
- **Bounded stewardship.** Stewards adjust only quantitative parameters — many of them bounded by hardcoded ranges (for example interest-rate maxima, fee-split floors, and the mint/redemption decay factors) — and cannot reach the core logic. See [Stewardship, not Governance](/stewardship).
- **Narrow oracle dependency.** Peg awareness uses internal redemption and minting volume rather than a pAsset price oracle. Collateral pricing still requires an ETH price feed; the provider, feed address, and network details are not published until launch.

For the full accounting of what can still go wrong, read the [Risk Disclosure](/resources/risk-disclosure).

## Bug bounty & responsible disclosure

A bug-bounty program and responsible-disclosure process are intended to run from launch. Until an official channel appears here, treat Polaris as having no public bounty and do not send exploit details to private forms, DMs, or third-party bounty pages.

Once live, this section will document:

- **Intake.** The single official channel for submitting a report.
- **Scope.** Which contracts, networks, and surfaces are in scope, and what is explicitly out of scope.
- **Severity tiers and rewards.** How findings are classified and what each tier pays.
- **Disclosure process.** The expected steps and timeline from report to fix to public disclosure, plus safe-harbor terms for good-faith research.

## Reporting a vulnerability

Use only the disclosure channel linked from this page once it is published. Until then, use public Polaris channels only to ask for the current intake path.
