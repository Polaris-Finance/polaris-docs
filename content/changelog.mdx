# Changelog

This is a changelog for **this documentation**, not the Polaris protocol. It records what changed in these pages — structure, content, and accessibility — so returning readers can see what is new.
Entries follow [Keep a Changelog](https://keepachangelog.com), use ISO dates (YYYY-MM-DD), and are listed newest first.

## [v0.1] - 2026-06-08

Initial Polaris documentation release for Public Testnet 1. This version establishes the app-task guides, protocol concept pages, and canonical reference pages needed to understand and safely test the system.

### Added

- Initial documentation structure for Introduction, Launch Status, Public Testnet Quickstart, Using the App, protocol mechanics, Resources, and Changelog.
- Public Testnet Quickstart, Troubleshooting, and Parameters pages.
- Expanded app-task coverage for Dashboard, Swap, Borrow, Manage Trove, Earn, Split, Zap, and Advanced.
- Point-of-decision risk notes on the action pages (open a trove, redemptions, Stability Pool deposits, conversions), each linking to the full Risk Disclosure.
- A POLAR tokenomics page covering distribution, allocations, and vesting.
- The Polaris system overview diagram on Core Concepts and the lineage diagram on Why Polaris, each with descriptive alt text, a caption, and a numbered text description so the meaning survives without the image.
- An `llms.txt` machine-readable export so agents can consume the docs programmatically.
- This documentation changelog.

### Changed

- Reorganized navigation around launch status, quickstart, app tasks, protocol concepts, and canonical references.
- Moved address ownership to Contracts & Addresses, parameter ownership to Parameters, and testnet artifact ownership to Public Testnet 1 Artifacts.
- Kept `/resources` as the glossary route; no dedicated category landing page was added.
- Enriched the six role paths from thin link lists into role-specific reading sequences, each with a "Who this is for" line, prerequisites, and a one-line note on why each step matters.
- Added audience signposting that routes users to the right starting point.
- Accessibility pass across the affected pages, including image alternatives and text descriptions for every embedded diagram.
- Trimmed over-long meta descriptions so search snippets don't truncate.
- Consolidated repeated cross-page explanations (conversion and floor mechanics, the stewardship "what it cannot do" list, Fee Router routing, the Recovery Mode interaction) so each concept is defined once and referenced elsewhere.
