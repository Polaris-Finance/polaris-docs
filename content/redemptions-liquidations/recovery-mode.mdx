# Recovery Mode

Recovery Mode is the protective state Polaris enters when system backing becomes thin relative to total pAsset debt.

It is not a shutdown and it does not freeze the system. It restricts actions that would make system health worse.

## The System Health Metric

Each pAsset market has a system-wide health metric that compares reserve backing to total debt.

When that ratio falls below the Recovery Mode threshold, the market enters Recovery Mode automatically.

Current thresholds live in [Parameters](/resources/testnet#parameters).

## What Changes

Recovery Mode does not change which positions are liquidatable.

A position is still liquidated only when its LTV exceeds the normal maximum.

What changes is the set of actions issuers can take.

Allowed actions:

- Add collateral.
- Repay debt.
- Make adjustments that improve or preserve system backing.

Restricted actions:

- Withdraw collateral.
- Issue more debt.
- Make adjustments that increase LTV or weaken backing.

In short: actions that make the system safer remain open. Actions that make the system riskier face tighter rules.

Recovery Mode does not automatically close healthy positions and does not change the normal liquidation threshold.

## How To Stay Safe

The defense is the same in every mode: keep a healthy LTV buffer.

To reduce risk:

1. Add pETH collateral.
2. Repay pAsset debt.
3. Avoid running positions close to the maximum LTV.
4. Build your buffer before volatile conditions arrive.

Recovery Mode is most likely during market stress, when adding collateral may be harder and execution may be more expensive.
