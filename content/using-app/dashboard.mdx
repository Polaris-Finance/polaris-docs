# Dashboard & Analytics

<PageStatusBanner lastUpdated="June 2026" />

The **Dashboard** is the app's orientation view. It is where you check balances, open positions, pending rewards, and system health before choosing a transaction tab. The **Analytics** view, covered further down, shows the protocol-wide metrics behind those numbers.

<Callout variant="info" title="Where this appears in the app">
App anchors: **Dashboard** for your own state, and **Analytics** for protocol-wide context. Use both for inspection only; execute actions from the matching task tab after checking a fresh quote.
</Callout>

*Each Earn card shows its APR, your deposited or staked amount, and the gains accrued so far across each reward token.*

## What to check

| Area | What it tells you |
|---|---|
| **Balances** | WETH, pETH, fpETH, vpETH, pAssets, POLAR, and claimable rewards held by the connected wallet |
| **Borrow positions** | Open troves, reserve loans, debt, collateral, and health metrics |
| **Earn positions** | Stability Pool deposits, POLAR Staking balances, and claimable gains |
| **System status** | Recovery Mode, floor ratio, borrow rates, pool size, and other launch/runtime signals |
| **Next actions** | Which tabs can adjust or close the position you are viewing |

## Read before acting

- A dashboard number is a state read, not a transaction quote.
- A position can change between the Dashboard and the moment your transaction lands.
- Use the transaction tab's quote and simulation before signing.

## Analytics: protocol-wide metrics

Use **Analytics** to inspect protocol-wide state before deciding on a transaction. Analytics is for context and monitoring; action tabs provide the final quote.

| Metric | Why it matters |
|---|---|
| **Floor price** | The contract-defined pETH floor under the bonding-curve assumptions |
| **Floor ratio** | pETH floor price divided by current pETH market price; frames curve-implied drawdown |
| **Borrow APR** | Annualized estimate of current pAsset debt cost |
| **Stability Pool size** | Available first-loss buffer for liquidations |
| **Recovery Mode status** | Whether a pAsset instance is under system-wide stress restrictions |
| **PSM/redemption activity** | Peg pressure and fee context for pAsset markets |
| **POLAR Staking rewards** | Current routed rewards where the staking surface is live |

Before relying on a metric:

- Check the timestamp or refresh state before acting.
- Distinguish an APR estimate from realized yield.
- Confirm whether a metric is testnet-only, illustrative, or production-published.
- Use the action tab's current quote for fees, slippage, and final amounts.
