# Polaris Safety and Verification

**Who this is for:** every Polaris user and integrator; run this path before connecting a wallet, sending ETH, minting a pAsset, depositing into a pool, locking POLAR, or integrating contracts.

**Prerequisites:** none. Complete this path *first*, before any other journey, because impersonation and phishing are most likely around launch and testnet phases.

> **Launch status:** [Polaris Launch Status](/launch-status) is the source of truth. As of June 8, 2026, public Public Testnet 1 is on Sepolia (`11155111`) and the official testnet app is [app.testnet.polarisfinance.io](https://app.testnet.polarisfinance.io/). Production Ethereum mainnet (`1`) is forthcoming, and no production app, production addresses, ABIs, verified source, final parameters, or audit reports are published yet.

> **Open Testnet App:** Use [https://app.testnet.polarisfinance.io/](https://app.testnet.polarisfinance.io/) only on Sepolia (`11155111`) with no-value test assets. Treat any production frontend, token, contract, airdrop, or support message as unverified until it is linked from [Launch Status](/launch-status).

<Callout variant="info" title="Where this appears in the app">
App anchors: **Dashboard** first for network and system state, then any action tab's pre-confirmation screen for contract, token, amount, fee, approval, and simulation checks.
</Callout>

## Verification checklist

Work top to bottom; each row names a check and where it will be confirmed at launch.

| Check | Where it will be confirmed |
|---|---|
| Current launch phase and network | [Polaris Launch Status](/launch-status) |
| Testnet app URL and production app status | [Polaris Launch Status](/launch-status) and [Getting Started](/getting-started) |
| Deployed addresses | [Launch Status](/launch-status), official Sepolia manifests, and [Contracts and Addresses](/resources/contracts) once production is live |
| Verified source and ABIs | [Contracts and Addresses](/resources/contracts), official manifests, and block explorer links |
| Audit reports | [Audits and Security](/resources/audits-security) |
| Final user parameters | The relevant product page and contract reference |
| Stewardship bounds | [Stewardship, not Governance](/stewardship/stewardship) |

## Things Polaris will not ask for

- A seed phrase or private key.
- KYC for protocol use.
- A direct transfer to a support wallet.
- Interaction with an unpublished address.
- Trust in a frontend that is not linked from official Polaris channels.

## Risk docs

Read [Risk Disclosure](/resources/risk-disclosure) for smart-contract, collateral, liquidation, peg, Stability Pool, pETH, stewardship, and phishing risks. Read [Audits and Security](/resources/audits-security) for the security model and audit status.

## Developer verification

For apps and indexers, add explicit allowlists for network IDs and contract addresses. Testnet allowlists should point to Sepolia (`11155111`) artifacts from [Launch Status](/launch-status) and the linked manifests. Production allowlists should stay empty until production addresses are published. Your integration should fail closed if the deployment manifest, ABI, or verified source does not match the official docs.
