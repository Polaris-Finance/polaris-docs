# Dashboard & Analytics

## What it does

The Dashboard is the app's read-only starting point. Use it to check wallet balances, open positions, claimable rewards, and system health before moving to Swap, Borrow, Earn, Split, Zap, or Advanced.

Analytics shows protocol-wide context such as pool sizes, borrow costs, floor-related metrics, and stress indicators. Treat Analytics as context, not as a transaction quote.

Image: Polaris Dashboard and Analytics showing wallet balances, open positions, rewards, and system-health cards

*The Dashboard is the read-only checkpoint before moving to action tabs.*

## Before you start

- Use Public Testnet 1 at [app.testnet.polarisfinance.io](https://app.testnet.polarisfinance.io).
- Connect to Sepolia, chain ID `11155111`.
- Confirm the frontend, network, and wallet prompts against [Safety & Verification](https://tokenbrice.github.io/polaris-docs/resources/safety-verification).
- Use test assets only. See [Public Testnet 1 Artifacts](https://tokenbrice.github.io/polaris-docs/resources/testnet).

## Steps/checks

1. Connect the wallet you want to inspect.
2. Check balances for WETH, pETH, fpETH, vpETH, pAssets, POLAR, and pending rewards.
3. Review borrow positions for debt, collateral, and health.
4. Review Earn positions for Stability Pool deposits, staking balances, and claimable gains.
5. Check system status before borrowing, repaying, withdrawing, or zapping.
6. Open Analytics for protocol-wide context, then return to the relevant action tab for a fresh quote.

A check worked when the connected wallet, Sepolia network, balances, positions, and timestamps match what you expect.

## Common failures

- Wallet is on the wrong chain.
- Balances do not appear because the token is not imported in the wallet.
- Dashboard data is stale; refresh before acting.
- APR or health values differ from a previous view because app state changed.
- A position is not shown because it belongs to another wallet.

## Risks/links

Dashboard values are state reads, not final transaction terms. Use the action tab quote before signing. For canonical status, risks, and addresses, use [Launch Status](https://tokenbrice.github.io/polaris-docs/launch-status), [Risk Disclosure](https://tokenbrice.github.io/polaris-docs/resources/risk-disclosure), and [Contracts & Addresses](https://tokenbrice.github.io/polaris-docs/resources/contracts).

## Next actions

- Use [Swap](https://tokenbrice.github.io/polaris-docs/using-app/swap) to get or exit pETH and pAssets.
- Use [Borrow](https://tokenbrice.github.io/polaris-docs/using-app/borrow) to open debt positions.
- Use [Manage Trove](https://tokenbrice.github.io/polaris-docs/using-app/manage-trove) to adjust, repay, or close existing troves.
- Use [Earn](https://tokenbrice.github.io/polaris-docs/using-app/earn) to deposit, withdraw, or claim.
- Use [Troubleshooting](https://tokenbrice.github.io/polaris-docs/troubleshooting) if wallet state, quotes, or balances look wrong.

# Advanced

## What it does

Advanced exposes tools that are closer to protocol operations than the guided app tabs. Use it for manual approvals, PSM actions, reserve-loan controls, conversion, staking controls, and contract checks only when those surfaces are available and you understand the action.

## Before you start

- Start from [Dashboard & Analytics](https://tokenbrice.github.io/polaris-docs/using-app) and verify the current app/network first.
- Prefer guided tabs when they support your action.
- Verify frontend, network, contract address, token, spender, amount, and function before signing.
- Keep [Contracts & Addresses](https://tokenbrice.github.io/polaris-docs/resources/contracts) and [Safety & Verification](https://tokenbrice.github.io/polaris-docs/resources/safety-verification) open.

## Steps/checks

1. Identify the exact action and contract.
2. Confirm the token, amount, spender, and receiving address.
3. Check whether a guided tab can do the same thing more safely.
4. Review wallet simulation and calldata warnings.
5. Sign only if the prompt matches your intent.
6. Confirm the resulting balance, allowance, position, or claim state.

The action worked when the intended state changed and no unexpected allowance or position remains.

## Common failures

- Wrong spender or stale approval.
- Wrong pAsset, token leg, or position owner.
- Action is unavailable in the current deployment.
- Manual transaction bypasses app prechecks.
- Wallet warning indicates an unexpected contract or permission.

## Risks/links

Advanced actions can bypass guardrails. Use [Safety & Verification](https://tokenbrice.github.io/polaris-docs/resources/safety-verification), [Contracts & Addresses](https://tokenbrice.github.io/polaris-docs/resources/contracts), [Risk Disclosure](https://tokenbrice.github.io/polaris-docs/resources/risk-disclosure), [Conversion Auctions](https://tokenbrice.github.io/polaris-docs/polar/conversion-auctions), and [POLAR Tokenomics](https://tokenbrice.github.io/polaris-docs/polar/tokenomics) as the canonical references.

## Next actions

- Return to [Dashboard & Analytics](https://tokenbrice.github.io/polaris-docs/using-app) to confirm state.
- Use the guided [Borrow](https://tokenbrice.github.io/polaris-docs/using-app/borrow), [Earn](https://tokenbrice.github.io/polaris-docs/using-app/earn), [Split](https://tokenbrice.github.io/polaris-docs/using-app/split), or [Swap](https://tokenbrice.github.io/polaris-docs/using-app/swap) page when possible.
- Use [Troubleshooting](https://tokenbrice.github.io/polaris-docs/troubleshooting) before retrying a failed manual transaction.

# Borrow

## What it does

Borrow is where you create collateralized positions. For pAssets, this means depositing pETH collateral and minting debt through a trove/CDP flow. Where exposed, Borrow may also include reserve-loan actions against fpETH.

Image: Polaris Borrow tab showing a pAsset trove form with collateral amount, debt amount, LTV, and risk details

*Borrow surfaces the position inputs and health checks before a trove action is signed.*

## Before you start

- Start from [Dashboard & Analytics](https://tokenbrice.github.io/polaris-docs/using-app) and verify the current app/network first.
- Hold the required collateral and gas.
- Understand that borrow positions can be affected by price movement, redemptions, liquidations, and Recovery Mode.
- Review [Open a Trove](https://tokenbrice.github.io/polaris-docs/minting/open-a-trove) for the full pAsset minting flow.

## Steps/checks

1. Choose the borrow asset or position type.
2. Enter collateral and debt amounts.
3. Review LTV, liquidation exposure, debt, fees, and projected position health.
4. Approve collateral if required.
5. Sign the borrow transaction.
6. Return to Dashboard and confirm debt, collateral, and health updated.

A borrow action worked when the position appears with the expected collateral, debt, and health metrics.

## Common failures

- LTV exceeds the allowed threshold.
- Debt amount is too small or too large for the current position.
- Recovery Mode or system restrictions block the action.
- Price movement changes projected health before inclusion.
- Approval targets the wrong token or spender.
- You are connected with a wallet that does not own the position.

## Risks/links

Borrowing creates liquidation and redemption exposure. Keep protocol explanations in [Minting pAssets](https://tokenbrice.github.io/polaris-docs/minting), [Redemptions](https://tokenbrice.github.io/polaris-docs/redemptions-liquidations), [Liquidations](https://tokenbrice.github.io/polaris-docs/redemptions-liquidations/liquidations), and [Recovery Mode](https://tokenbrice.github.io/polaris-docs/redemptions-liquidations/recovery-mode). Use [Parameters](https://tokenbrice.github.io/polaris-docs/resources/parameters) for current values and [Risk Disclosure](https://tokenbrice.github.io/polaris-docs/resources/risk-disclosure) for risks.

## Next actions

- Use [Manage Trove](https://tokenbrice.github.io/polaris-docs/using-app/manage-trove) to adjust, repay, or close the position.
- Use [Swap](https://tokenbrice.github.io/polaris-docs/using-app/swap) to acquire or rebalance assets.
- Use [Earn](https://tokenbrice.github.io/polaris-docs/using-app/earn) if you want to deposit pAssets into the Stability Pool.

Relevant app/search vocabulary: Borrow, Manage Trove, borrow, mint, open trove, repay, close trove, pUSD, pGOLD, LTV, loan-to-value, official app.

# Earn

## What it does

Earn groups protocol-native yield actions exposed in the app. This includes Stability Pool deposits, withdrawals, claimable gains, and POLAR staking controls where available in the current deployment.

Image: Polaris Earn tab showing Stability Pool deposits, staking controls, claimable rewards, and APR context

*Earn groups deposit, withdrawal, claim, and staking actions in one place.*

## Before you start

- Start from [Dashboard & Analytics](https://tokenbrice.github.io/polaris-docs/using-app) and verify the current app/network first.
- Hold the deposit token and Sepolia ETH for gas.
- Check whether you are depositing into a Stability Pool, staking POLAR, or claiming rewards.
- Read [Stability Pool](https://tokenbrice.github.io/polaris-docs/yield) before depositing.

## Steps/checks

1. Select the Earn surface.
2. Check deposit balance, current deposit, APR estimate, and pending gains.
3. Choose deposit, withdraw, claim, or stake/unstake.
4. Review whether the action also claims pending rewards.
5. Approve the token if required.
6. Sign the transaction.
7. Confirm Dashboard balances, deposits, and claimable gains updated.

The action worked when the Earn card and wallet balances reflect the transaction.

## Common failures

- Deposit token balance is too low.
- Approval was signed but the action transaction was not.
- Pending rewards are dust-sized or not claimable yet.
- Withdrawal or claim is disabled by current app state.
- APR changed after refresh.
- Wallet is connected to the wrong account.

## Risks/links

Stability Pool deposits are first-loss capital during liquidations. APR estimates are not guaranteed returns. Use [Yield Sources](https://tokenbrice.github.io/polaris-docs/yield/yield-sources), [POLAR Tokenomics](https://tokenbrice.github.io/polaris-docs/polar/tokenomics), and [Risk Disclosure](https://tokenbrice.github.io/polaris-docs/resources/risk-disclosure) for the underlying risk model.

## Next actions

- Use [Swap](https://tokenbrice.github.io/polaris-docs/using-app/swap) to acquire deposit assets.
- Use [Zap](https://tokenbrice.github.io/polaris-docs/using-app/zap) for bundled swap-plus-deposit.
- Use [Dashboard & Analytics](https://tokenbrice.github.io/polaris-docs/using-app) to monitor deposits and rewards.

Relevant app/search vocabulary: Earn, earn, yield, APR, Stability Pool, deposit, claim rewards.

# Manage Trove

## What it does

Manage Trove is the ongoing maintenance flow for an existing pAsset debt position. Use it to add collateral, mint more, repay debt, withdraw collateral when allowed, close the trove, or inspect liquidation exposure.

## Before you start

- Start from [Dashboard & Analytics](https://tokenbrice.github.io/polaris-docs/using-app) and verify the current app/network first.
- Connect the wallet that owns the trove.
- Check the current LTV, debt, collateral, borrow rate, and Recovery Mode state.
- Use [Parameters](https://tokenbrice.github.io/polaris-docs/resources/parameters) for current thresholds and [Managing Your Trove](https://tokenbrice.github.io/polaris-docs/minting/managing-your-trove) for mechanics.

## Steps/checks

1. Open the existing trove from Dashboard or Borrow.
2. Choose the action: add collateral, mint more, repay, withdraw collateral, or close.
3. Review the resulting LTV and whether the action increases or reduces risk.
4. Confirm allowances if the action spends pETH or pAssets.
5. Sign the transaction.
6. Return to Dashboard and confirm collateral, debt, and LTV updated.

The action worked when the trove state matches the intended collateral and debt change.

## Common failures

- The connected wallet does not own the trove.
- Repay amount exceeds your pAsset balance.
- Withdraw or mint would push LTV above the required threshold.
- Recovery Mode blocks risk-increasing actions.
- Interest accrued between quote and inclusion.
- A redemption changed the trove before your transaction landed.

## Risks/links

A low LTV protects against liquidation, not against proportional redemptions. Use [Redemptions](https://tokenbrice.github.io/polaris-docs/redemptions-liquidations), [Liquidations](https://tokenbrice.github.io/polaris-docs/redemptions-liquidations/liquidations), [Recovery Mode](https://tokenbrice.github.io/polaris-docs/redemptions-liquidations/recovery-mode), and [Risk Disclosure](https://tokenbrice.github.io/polaris-docs/resources/risk-disclosure) for the risk model.

## Next actions

- Use [Borrow](https://tokenbrice.github.io/polaris-docs/using-app/borrow) to open another pAsset position.
- Use [Swap](https://tokenbrice.github.io/polaris-docs/using-app/swap) to acquire pETH or repayment assets.
- Use [Dashboard & Analytics](https://tokenbrice.github.io/polaris-docs/using-app) to monitor the position before markets move.

Relevant app/search vocabulary: Borrow, Manage Trove, borrow, mint, open trove, repay, close trove, pUSD, pGOLD, LTV, loan-to-value, official app.

# Split

## What it does

Split converts pETH into matching fpETH and vpETH balances. Merge reverses the action when you hold equal amounts of both legs. Use this page for the app workflow; use [pETH Split](https://tokenbrice.github.io/polaris-docs/peth/split) for the mechanism.

Image: Polaris Split tab showing pETH split and merge controls for fpETH and vpETH balances

*Split and Merge change pETH exposure, so review both legs before signing.*

## Before you start

- Start from [Dashboard & Analytics](https://tokenbrice.github.io/polaris-docs/using-app) and verify the current app/network first.
- Hold pETH for splitting, or equal fpETH and vpETH amounts for merging.
- Confirm which leg you intend to hold after the action.

## Steps/checks

1. Choose Split or Merge.
2. Enter the amount.
3. Review the expected fpETH, vpETH, or pETH output.
4. Approve pETH, fpETH, or vpETH if required.
5. Sign the transaction.
6. Confirm wallet balances update.

A split worked when pETH decreases and fpETH plus vpETH increase by the expected matching amount. A merge worked when equal fpETH and vpETH decrease and pETH increases.

## Common failures

- Merge amount exceeds the smaller of your fpETH or vpETH balances.
- Token balance exists but is hidden in the wallet UI.
- Approval was granted for one leg but not the other.
- Transaction fails because gas or allowance is too low.
- You expected holding one leg to behave like holding unsplit pETH.

## Risks/links

Holding only fpETH or only vpETH changes your exposure. Read [pETH Split](https://tokenbrice.github.io/polaris-docs/peth/split), [pETH Floor Price](https://tokenbrice.github.io/polaris-docs/peth/floor-price), and [Risk Disclosure](https://tokenbrice.github.io/polaris-docs/resources/risk-disclosure) before relying on either leg.

## Next actions

- Use [Borrow](https://tokenbrice.github.io/polaris-docs/using-app/borrow) if reserve-loan actions are exposed for fpETH.
- Use [Swap](https://tokenbrice.github.io/polaris-docs/using-app/swap) if the app quotes a supported leg route.
- Use [Advanced](https://tokenbrice.github.io/polaris-docs/using-app/advanced) only for manual actions you understand.

Relevant app/search vocabulary: Swap, swap, Split, fpETH, vpETH, pETH, floor price.

# Swap

## What it does

Swap exchanges app-supported assets through the route quoted by Polaris. Routes may include WETH, pETH, fpETH, vpETH, pAssets, or protocol modules exposed by the current Public Testnet 1 deployment.

Image: Polaris Swap tab showing input and output assets, quoted route, minimum received, and slippage controls

*Swap quotes should be refreshed and reviewed before approval or execution.*

## Before you start

- Start from [Dashboard & Analytics](https://tokenbrice.github.io/polaris-docs/using-app) and verify the current app/network first.
- Hold the input token and enough Sepolia ETH for gas.
- Read the route, output, minimum received, price impact, and approval target before signing.

## Steps/checks

1. Choose the input and output assets.
2. Enter the amount.
3. Review the route and quote.
4. Check minimum received, price impact, fees, and slippage.
5. Approve the token only if the spender matches the app flow.
6. Sign the swap transaction.
7. Confirm the output balance appears in the wallet and Dashboard.

The swap worked when the transaction lands and the output token balance updates.

## Common failures

- Quote expired before the transaction landed.
- Slippage limit was too tight.
- Input balance or allowance was too low.
- Gas was too low or Sepolia was congested.
- Wallet is connected to the wrong account or network.
- The route changed after refresh.

## Risks/links

Swaps can lose value through slippage, price movement, stale quotes, gas, and execution ordering. pETH and pAsset mechanics are explained in [pETH](https://tokenbrice.github.io/polaris-docs/peth), [pETH Floor Price](https://tokenbrice.github.io/polaris-docs/peth/floor-price), and [pAsset Catalog](https://tokenbrice.github.io/polaris-docs/minting/passet-catalog). Use [Risk Disclosure](https://tokenbrice.github.io/polaris-docs/resources/risk-disclosure) for risk details.

## Next actions

- Use [Borrow](https://tokenbrice.github.io/polaris-docs/using-app/borrow) after acquiring pETH or a pAsset.
- Use [Earn](https://tokenbrice.github.io/polaris-docs/using-app/earn) after acquiring a Stability Pool asset.
- Use [Zap](https://tokenbrice.github.io/polaris-docs/using-app/zap) if you want the app to bundle swap plus deposit.

Relevant app/search vocabulary: Swap, swap, Split, fpETH, vpETH, pETH, floor price.

# Zap

## What it does

Zap bundles multiple app actions into a guided route. A common route is swapping WETH into a pAsset and depositing that pAsset into the Stability Pool. The zap is an execution shortcut; it does not change the economics of the underlying steps.

Image: Polaris Zap tab showing the pUSD Stable Yield route from WETH through pUSD into the Stability Pool

*The pUSD Stable Yield zap routes WETH into pUSD, then deposits the pUSD into the Stability Pool in one guided flow.*

## Before you start

- Start from [Dashboard & Analytics](https://tokenbrice.github.io/polaris-docs/using-app) and verify the current app/network first.
- Hold the input asset and gas.
- Read the full route before signing.
- Be prepared to finish manually through Swap or Earn if a bundled step fails.

## Steps/checks

1. Choose the zap goal.
2. Enter the input amount.
3. Review every route leg, output estimate, slippage, deposit target, and approval.
4. Simulate or refresh the quote if the app offers it.
5. Sign approvals only for expected spenders.
6. Sign the zap transaction or transaction sequence.
7. Confirm final balances and Earn deposits on Dashboard.

The zap worked when the final target state appears, not merely when the first approval succeeds.

## Common failures

- One leg became stale before inclusion.
- Slippage or liquidity changed.
- Approval succeeded but the bundled action failed.
- Gas estimate was too low for the full route.
- The wallet now holds an intermediate asset instead of the final deposit.

## Risks/links

A zap combines swap, approval, and deposit risks. If a zap stops midway, inspect balances before retrying. Use [Swap](https://tokenbrice.github.io/polaris-docs/using-app/swap), [Earn](https://tokenbrice.github.io/polaris-docs/using-app/earn), [Stability Pool](https://tokenbrice.github.io/polaris-docs/yield), and [Troubleshooting](https://tokenbrice.github.io/polaris-docs/troubleshooting) for manual fallback.

## Next actions

- Use [Dashboard & Analytics](https://tokenbrice.github.io/polaris-docs/using-app) to confirm the final state.
- Use [Swap](https://tokenbrice.github.io/polaris-docs/using-app/swap) if you received an intermediate token.
- Use [Earn](https://tokenbrice.github.io/polaris-docs/using-app/earn) to complete a deposit manually.

Relevant app/search vocabulary: Earn, earn, yield, APR, Stability Pool, deposit, claim rewards.
